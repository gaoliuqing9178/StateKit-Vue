# 这轮已经完成的事情

## 1. `0.2.1` 发布已经完成

- npm 上最新已发布版本是 `@statekit-vue/shared@0.2.1` 和 `@statekit-vue/vue@0.2.1`。
- 本地 workspace 仍与 `0.2.1` 版本线保持一致。

当前推荐的公开入口是：

- `EmptyState`
- `OnboardingState`
- `LoadingState`
- `ErrorState`
- `PermissionState`
- `UpgradeState`
- `SuccessState`

底层仍然保留 19 个 preset recipes。旧的场景名导出依然存在，但已经作为 deprecated compatibility exports 处理。

## 2. onboarding category pilot 已落地（未发版）

- 这轮已经把 onboarding 从 empty 中拆出，作为独立 category 对外提供 `OnboardingState`。
- shared 元数据新增了 `onboarding-workspace`，用于承载"首次创建工作区"的首个试点场景。
- `first-project` 没有删除，但现在更适合表达"工作区已经存在，只是还没有第一个项目"的 empty-state 过渡语义。
- docs 首页、recipes 列表、recipe detail、docs 示例页和 `examples/vite-vue-admin` 都已经同步到 7 类 / 19 recipes 的新口径。

## 3. OnboardingActivationView 新增（未发版）

- 新增 `apps/docs/src/views/examples/OnboardingActivationView.vue`，展示完整 first-run 激活链路：创建 workspace → 邀请成员 → 连接集成 → 完成。
- 新增 `/examples/onboarding-activation` 路由，`/examples` 默认跳转至此。
- 这是 docs 里第一个端到端串联多个 category 的流程型示例页，使"onboarding 不是 generic empty state"这条语义差异有了可视化产品上下文。

## 4. Playwright 测试覆盖全面扩充（未发版）

新增了以下 spec 文件：

- `apps/docs/tests/home-featured-recipes.spec.ts` — 首页精选 recipe 展示
- `apps/docs/tests/installation-navigation.spec.ts` — 安装引导页导航
- `apps/docs/tests/mobile-layout.spec.ts` — 全主路径移动端断点回归
- `apps/docs/tests/onboarding-activation.spec.ts` — OnboardingActivationView 端到端流程
- `apps/docs/tests/recipe-detail-paths.spec.ts` — recipe 详情页路径参数

合并已有的 `admin-empty-states.spec.ts`、`home-navigation.spec.ts`、`permissions-and-upgrade.spec.ts`、`recipes-navigation.spec.ts`、`task-flow.spec.ts`，当前共 10 个 Playwright spec 文件，覆盖所有主路径和移动端布局。

## 5. `examples/vite-vue-admin` 重做为 onboarding-to-completion 流程（未发版）
